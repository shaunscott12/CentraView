About WebForm Import

Webform Import Instructions

---------------------------

In order for an email to have the ability to be imported into CentraView

as a Contact, the following Header must be set on the email when it is

sent:

X-CentraView-Webform: Contact

This should be one of the standard RFC-822 mail headers at the top of

the message source. Some mail clients have the ability of setting an

outgoing header on a message. Also, most programming languages offer the

ability to add a header to a message before it is sent. This is how we

have intended Webform Import to work with CentraView.

A developer would create a form on a webpage which submits to a script

written in any programming language with the ability to send email.

(PHP, perl, JSP, Java, ASP, .Net, etc) The programmer would have to

write a script to take input from the web form, and send an email to a

given email address. That given email address can then be checked within

CentraView's Email module. When the user views the message with the

X-CentraView-Webform: Contact header set, a button will appear at the

top of the screen, allowing the user to import the content of the

message as a Contact record.

The email must be formatted within the proper guidelines in order to be

validly imported into CentraView. The most critical issue is that the

fields must have the correct names and the correct format. The field

names are case sensitive and must appear at the beginning of the line,

followed by a colon, followed by a space, and finally, the content to be

imported into that field.

For example, to import "John" into the First Name field, the email would

need to contain the line:

First Name: John

Again, the field name (all content before the ":") must be exactly as

specified in the field list below. Custom fields are also able to be

imported. They cannot be listed below, due to their dynamic nature. In

order to a Custom Field to be imported, it must be configured in

CentraView first; meaning that an Administrative user must create the

Custom Field in the Administration module.

Any fields listed in the email, which are not valid fields, will not be

imported. They will not adversely affect the functionality of Webform

Import, they will simply be ignored.

The email can also contain content other than the fields to be imported.

This content can appear anywhere within the email message, it will

simply be ignored.

Here is an example of a valid import email:

-----BEGIN EMAIL------

Someone has submitted a form on yourwebsite.com. Their information is

below.

First Name: John

Last Name: Doe

Main Phone: 555-555-1234

Fax Phone:

Email: example@example.com

Industry: Sales and Marketing

Entity Name: MyCompany, Corp.

Entity Main Phone: 555-345-2222

Company Home: Washington

This email can be imported using the webform import utility.

------END EMAIL-------

In the example above, the following fields will be imported correctly:

-First Name

-Last Name

-Main Phone

-Fax Phone (empty)

-Email

-Industry (if set up as a Custom Field)

-Entity Name

-Entity Main Phone

The field "Company Home" will be ignored, unless it was set up as a

custom field. All other lines in the message will be ignored by the

Webform Import.

One thing to note is that all fields pertaining to an Entity record will

have "Entity " as the first word in the field name. So you may have the

Custom Field "Industry" set up for both Entity records and Individual

records. To allow both fields to work through Webform Import, you would

place "Entity " before "Industry" to indicate that this is the Industry

field related to the Entity, and not the Individual.

Once the email has reached the Inbox of CentraView's Email module, the

user can view the message details. On the message details screen, a

button will appear at the top of the page labeled "Import Contact".

When the user clicks this button, a screen will open showing the fields

in the email, along with the values from the email. The user has the

option to change any of these values at this time. Only valid fields

will be shown on this screen.

When the user is done modifying any of the values they wish, they will

click "Save" to save the Contact into the database. At this time, the

process is complete.
